# 低保真原型设计指南

## 什么是低保真原型

低保真原型（Low-Fidelity Prototype）是使用简单线条、方框和文字，不加视觉修饰的原型。用于快速验证想法、收集反馈，不用于展示给客户。

## 何时使用

- 设计早期，想法尚未确定时
- 需要快速测试多个方案时
- 团队内部评审，不需要精美效果时
- 探索信息架构和交互逻辑时

## 核心要素

### 页面结构
```
┌─────────────────────────┐
│        Header           │  ← 标题栏/导航
├─────────────────────────┤
│                         │
│       Content           │  ← 主要内容区
│                         │
├─────────────────────────┤
│        Footer           │  ← 底部/操作栏
└─────────────────────────┘
```

### 常用组件符号

| 组件 | 符号 |
|------|------|
| 文字输入框 | ┌──────────────┐ |
| 按钮 | [ 按钮文字 ] |
| 图片/视频 | ┌──┐ |
| 复选框 | ☐ 选项文字 |
| 单选按钮 | ○ 选项文字 |
| 下拉菜单 | ▼ 选择 ▼ |
| 页面链接 | ──→ 目标页面名 |

### 交互标注

在低保真原型上用箭头和编号标注交互：

```
[按钮] --①--> [弹窗]
      --②--> [跳转至详情页]
```

## 工具推荐

### 数字工具
- **Figma**（免费，支持协作，推荐）
- ** Balsamiq**（专门做低保真，有手绘风格）
- **Sketch**（macOS专用）
- **Pixso**（国产，免费，功能接近Figma）

### 纸笔工具
- A4纸 + 马克笔
- 便利贴（不同颜色代表不同页面/状态）

## 页面优先级排序

先做核心页面，优先级：

1. **核心任务页** — 用户完成主要目标的页面
2. **关键流程页** — 主流程必经之路
3. **异常状态页** — 空状态、加载状态、错误状态
4. **辅助页面** — 设置、关于、帮助等

## 信息架构原则

```
一级导航（Tab或侧边栏）
  ├── 二级页面A
  │     ├── 三级详情页A1
  │     └── 三级详情页A2
  ├── 二级页面B
  └── 二级页面C
```

层级不超过3级，单个导航项不超过7个。

## 常见反模式

- ❌ 一个页面塞满所有功能（分而治之）
- ❌ 跳过低保真直接做高保真（先验证再投资）
- ❌ 用高保真收集用户反馈（引导性太强）
- ❌ 忽略异常状态（空数据、加载、网络错误）
