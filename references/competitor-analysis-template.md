# 竞品分析模板

## 竞品分析的目的

1. 了解市场现状与竞争格局
2. 发现行业最佳实践（Best Practices）
3. 找到差异化机会点
4. 避免重复造轮子

## 分析框架

### 1. 竞品选择

| 竞品名称 | 类型 | 选择理由 |
|---------|------|---------|
| 竞品A | 直接竞品（目标用户相同） | 市场份额大 |
| 竞品B | 直接竞品 | 功能创新 |
| 竞品C | 间接竞品（目标用户不同） | 有独特思路 |

建议选择3-5个竞品，兼顾头部产品和创新型产品。

### 2. 功能对比矩阵

| 功能模块 | 竞品A | 竞品B | 竞品C | 我们 |
|---------|-------|-------|-------|------|
| 首页推荐 | ✅ | ✅ | ❌ | ？ |
| 搜索过滤 | ✅ | ❌ | ✅ | ？ |
| 社交分享 | ✅ | ✅ | ✅ | ？ |
| 消息通知 | ✅ | ❌ | ❌ | ？ |

✅ = 有此功能且体验好  
⚠️ = 有但体验差  
❌ = 没有

### 3. 体验五维度对比

| 维度 | 竞品A | 竞品B | 竞品C |
|------|-------|-------|-------|
| **有用性**：功能是否解决核心需求 | | | |
| **可用性**：学习成本、操作效率 | | | |
| **可触性**：视觉设计、品牌感知 | | | |
| **情感性**：使用过程中的愉悦感 | | | |
| **可靠性**：稳定性、错误率 | | | |

评分：⭐-⭐⭐⭐⭐⭐

### 4. 用户评价分析

**应用商店评分**：
| 应用 | App Store | Android |
|------|-----------|---------|
| 竞品A | 4.2星 | 4.0星 |
| 竞品B | 4.5星 | 4.3星 |

**典型用户评论**：
| 来源 | 好评关键词 | 差评关键词 |
|------|----------|---------|
| 竞品A | "界面简洁" | "广告太多" |
| 竞品B | "功能齐全" | "太复杂" |

### 5. 商业模式对比

| 竞品 | 盈利模式 | 定价策略 | 优势 |
|------|---------|---------|------|
| 竞品A | 广告+会员 | freemium | 用户基数大 |
| 竞品B | 付费下载 | 一次性$9.9 | 收入稳定 |

## 洞察提炼

### SWOT分析

| | **有帮助** | **无帮助** |
|---|---|---|
| **内部** | Strength（优势） | Weakness（劣势） |
| **外部** | Opportunity（机会） | Threat（威胁） |

### 差异化机会

| 机会点 | 现有竞品处理方式 | 我们的机会 |
|--------|-----------------|---------|
| 竞品普遍未做智能推荐 | 仅按类别展示 | AI个性化推荐 |

## 分析注意事项

1. **客观中立**：避免先入为主，每条结论有事实依据
2. **聚焦重点**：不需要覆盖所有功能，聚焦核心差异
3. **持续更新**：竞品分析不是一次性工作，定期更新
4. **用户视角**：结合用户评价，而非只看功能清单
